# Archive Main Index

这是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定网站或域名，仅作为静态 HTML 资源的集中存放与索引。

## 项目简介

本仓库主要收录各类独立的 HTML 页面，可能包括但不限于信息展示页、工具页面、存档页面等。  
所有页面均为自包含的 HTML 文件，不依赖外部服务器环境，可直接在浏览器中打开预览。

## 目录结构

```
archive-main-index/
├── index.html          # 主索引页面，列出所有归档页面入口
├── pages/              # 归档的 HTML 页面存放目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
└── README.md           # 本说明文件
```

- `index.html`：仓库的主入口文件，用于快速浏览和跳转到各个归档页面。
- `pages/`：所有归档的 HTML 页面均放置在此目录下，便于管理和维护。

## 页面归档说明

归档的 HTML 页面通常具有以下特点：

- 每个页面均为独立的、完整的 HTML 文档。
- 页面中不包含动态后端逻辑，仅依赖前端技术（HTML、CSS、JavaScript）。
- 页面内容可能包含文本、图片、表格、链接等常见网页元素。
- 部分页面可能引用外部资源（如字体、图片、样式库），但这些资源通常托管于公共 CDN 或与页面一同归档。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 用浏览器打开 `index.html`，即可看到所有归档页面的索引。
3. 点击索引中的链接，即可查看对应的 HTML 页面。

你也可以直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件。

## 维护说明

- 新增归档页面时，请将 HTML 文件放入 `pages/` 目录，并更新 `index.html` 中的索引列表。
- 建议保持页面文件命名简洁、有序，便于长期管理。
- 若页面引用了外部资源，请确保资源链接有效，或考虑将资源一并归档。
- 本仓库不保证所有页面的内容永久有效或更新，请自行评估使用。
